# chez-xml λ-0.2 Design

## Scope

Three capabilities added to the v0.1 lite XML parser, plus one bug fix.  
All changes in `src/xml.sls` (~30 lines net). No new deps, no new files.

## 1. Bug fix: `parse-comment` error call

**Current:** `("Parser Error: invalid comment")` — string literal in function-call position.
Runtime crash: "attempt to apply non-procedure".

**Fix:** `(error "Parser Error: invalid comment")`

## 2. DTD skip (`<!DOCTYPE ...>`)

**Current behavior:** `parse-struct` sees `<!` not followed by `-` → delegates to `parse-decl`, but only when `class='head'`. `parse-decl` naively consumes chars until `>`, which breaks on internal subsets containing `>` (e.g. `<!DOCTYPE root [<!ELEMENT root ANY>]>`).

**v0.2 behavior:** After `<!`, check if it's a DOCTYPE declaration (case-insensitive prefix match). If yes, enter brace-depth tracking that skips to the matching `>`; if no, fall through to the existing `parse-decl` path. The head-only gate is removed — DTDs anywhere are quietly consumed.

**Implementation outline:**
- `parse-struct`: after consuming `!`, peek at the DOCTYPE keyword. Match → `parse-dtd`. No match → existing `parse-decl` (head gate removed).
- `parse-dtd`: depth-counter loop. `[` +1, `]` -1, terminate on `>` when depth=0.

## 3. Comments inside tags

**Current behavior:** `attrs-loop` and `label-loop` error when they encounter `<`. Real XML allows `<!-- ... -->` inside open-tag bodies.

**v0.2 behavior:** Both loops detect `<` + `!` + `-` + `-` and delegate to `parse-comment`, then continue the parent loop.

**Implementation outline:**
- `attrs-loop`: add `#\<` case. Peek 3 chars; if `! - -`, push them back (via `ch-buffer`) and call `parse-comment`, then recurse.
- `label-loop`: same pattern.

## 4. Declaration / xml-head anywhere

**Current behavior:** `<?xml ...?>` and `<! ... >` (non-comment, non-DOCTYPE) only recognized when `class='head'`. Else error.

**v0.2 behavior:** Remove the head-only gates. `<?` at any position → `parse-xml-decl`. `<!` (non-comment, non-DOCTYPE) at any position → `parse-decl`.

**Implementation outline:** Drop the `eq? class 'head` guards and their error else-branches.
